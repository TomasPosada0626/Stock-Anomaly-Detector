# Production Deployment Guide

This guide covers production-oriented deployment controls for QuantVision.

## Architecture Recommendation
- Streamlit service for user-facing UI.
- FastAPI service for API endpoints.
- Dedicated scheduler worker process/container.
- PostgreSQL as primary datastore.
- Optional Redis for queueing/caching extension.

## Required Environment Variables
- `USE_SQLALCHEMY_REPOSITORIES=true`
- `DATABASE_URL=postgresql+psycopg://<user>:<password>@<host>:5432/<db>`
- `SCHEDULER_WORKER_MODE=true`
- `SCHEDULER_INTERVAL_MINUTES=15`
- `SCHEDULER_MAX_CONSECUTIVE_FAILURES=10`
- `SCHEDULER_HEARTBEAT_FILE=storage/logs/scheduler_heartbeat.json`

## Security Hardening
- Keep API security headers and rate limiting enabled.
- Use strong secrets for all credentials and tokens.
- Do not hardcode `DATABASE_URL` in source files.
- Restrict ingress to HTTPS only.

## Database Initialization
1. Deploy database.
2. Set `DATABASE_URL`.
3. Run:
   - `python scripts/bootstrap_postgres.py`

## Backup Automation
Use:
- `scripts/backup_postgres.sh`

Required environment:
- `PGPASSWORD`
- optional: `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `BACKUP_DIR`, `RETENTION_DAYS`

Example:

```bash
export PGPASSWORD="***"
bash scripts/backup_postgres.sh
```

## Release Process
A manual semantic-version workflow is available:

Kubernetes manifests:
- `deploy/k8s/api-deployment.yml`
- `deploy/k8s/ui-deployment.yml`

Required secret:
- `DATA_ENCRYPTION_KEY` must be provided via your secret manager or Kubernetes Secret.

From GitHub Actions, choose bump type:
- patch
- minor
- major

The workflow creates and pushes a tag, then publishes a GitHub Release with autogenerated notes.

## Post-Deploy Checks
1. Verify `/health` and `/health/detailed`.
2. Verify metrics endpoint `/metrics`.
3. Run login + anomaly flow smoke test.
4. Confirm scheduler heartbeat updates on cycle.
